Introduction

Notre projet est composé de deux versions programmées en Java, chacune visant à fournir une solution robuste et conviviale pour l'analyse et l'amélioration de codes Python. La première version se présente sous la forme d'une application console, offrant une interface en ligne de commande (CLI) permettant une utilisation simple et efficace en mode terminal. La seconde version, quant à elle, adopte une approche graphique grâce à l'utilisation de la bibliothèque Swing. Cette interface graphique (GUI) offre une expérience utilisateur plus visuelle et interactive, facilitant la navigation à travers les fichiers Python.

Ces deux versions offrent une solution complète pour analyser les fichiers Python, détecter les annotations de type manquantes et la documentation, tout en fournissant des statistiques quantitatives sur la qualité du code.

Objectifs

Les objectifs principaux sont les suivants :

  • Lister les fichiers python dans un répertoire avec leurs informations principales (taille du fichier en Ko, date du fichier).
  • Analyser des fichiers Python pour détecter les annotations de type manquantes.
  • Vérifier la présence des commentaires d'en-tête dans les fichiers Python.
  • Identifier les fonctions sans commentaires "PyDoc" dans les fichiers Python
  • Fournir des statistiques qualitatives sur la qualité du code.
